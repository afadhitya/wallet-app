## Architecture Overview

```
cmd/wallet/main.go          (entry point)
        │
        ▼
internal/cli/               (presentation layer — Cobra commands)
        │
        ▼
internal/service/           (business logic layer — orchestration, validation)
        │
        ▼
internal/gen/               (data access layer — sqlc-generated Querier interface)
        │
        ▼
internal/db/                (infrastructure — SQLite connection, migrations)
```

### Layer Responsibilities

- **`cmd/wallet/main.go`** — Entry point only. Creates and executes the root Cobra command.
- **`internal/cli/`** — Cobra commands. Each Go file handles one command group (e.g., `add.go`, `budget.go`, `bill.go`). `helpers.go` wires services, handles JSON output encoding, classifies errors into machine-readable codes.
- **`internal/service/`** — Business logic. Each Go file handles one domain (e.g., `transaction.go`, `budget.go`, `forecast.go`). All database access goes through the `gen.Querier` interface (never direct SQL in service layer).
- **`internal/gen/`** — Auto-generated by sqlc. `models.go` has all model structs. `querier.go` defines the 92-method `Querier` interface used for mocking in tests. Never edit files in this directory.
- **`internal/db/`** — SQLite connection opening, WAL journal mode, migration application. Migrations are `.sql` files embedded via `go:embed`.
- **`pkg/config/`** — TOML config loading for main config and exchange rates.

### Key Patterns

- **Service wiring**: `withService()` in `helpers.go` creates the service by loading config, opening DB, and running migrations. Tests override via `getServiceOverride` function variable.
- **JSON output**: Global `--json` persistent flag. Success: `{"success": true, "data": {...}}`. Error: `{"success": false, "error": {"code": "...", "message": "..."}}`.
- **Error handling**: `NotFoundError` and `ValidationError` wrapper structs plus sentinel errors (`ErrInvalidAmount`, `ErrDuplicateName`, etc.). `classifyError()` maps error types to stable error codes (e.g., `CATEGORY_NOT_FOUND`, `INVALID_AMOUNT`, `BILL_PAUSED`).
- **Testability**: Service accepts `gen.Querier` interface for mocking. Config loading and rate config use package-level variables that tests can override. CLI tests inject `Service` via `getServiceOverride`.

## Database

### Engine

SQLite via `modernc.org/sqlite` (pure Go, no CGO). WAL journal mode for file databases. In-memory (`:memory:`) for tests. Foreign keys enabled.

### Tables

| Table | Purpose |
|-------|---------|
| `accounts` | Account with name, type, currency, balance, sort_order |
| `categories` | 2-level hierarchy with 32 system-seeded categories |
| `tags` | Freeform labels with unique name constraint |
| `transactions` | Core entity: expense, income, transfer, adjustment |
| `transaction_tags` | M2M junction: transactions ↔ tags |
| `budgets` | Snapshot-per-period budget with recurrence and notification threshold |
| `budget_categories` | M2M junction: budgets ↔ categories |
| `budget_tags` | M2M junction: budgets ↔ tags |
| `planned_payments` | Recurring bills with RRULE support, lifecycle state |
| `exchange_rates` | Currency conversion rates |
| `schema_version` | Migration version tracking |

### Migrations

New migrations go in `internal/db/migrations/NNN_description.sql`. Applied sequentially — already-applied migrations are skipped. The `schema_version` table tracks state.

### Soft Delete

- `accounts.is_archived`, `categories.is_archived`, `transactions.is_archived` — set to 1 instead of hard delete
- `budgets.is_active` — set to 0 instead of delete
- `planned_payments.is_active` — set to 0 for archived bills; hard deleted when one-time bill is paid
- All list/balance queries filter `WHERE is_archived = 0` / `WHERE is_active = 1`

## Transaction Types

| Type | Description |
|------|-------------|
| `expense` | Spending money. Requires category. Negative impact on account balance. |
| `income` | Receiving money. Requires category. Positive impact on account balance. |
| `transfer` | Moving money between own accounts. Uses `transfer_to_id` FK. Single-row model — balance query uses CASE to subtract from source and add to destination. |
| `adjustment` | Balance correction. `category_id` is NULL, no tags. Excluded from all reports. Created automatically by `wallet adjust`. |

## Amounts

**All amounts are `int64` in integer minor units.** Never use `float64` for amounts.

- IDR: `50000` = Rp 50,000
- USD: `5000` = $50.00
- JPY: `500` = ¥500

For multi-currency transactions, `base_amount` stores the converted value in the base currency. Display formatting adds thousand separators: `50000` → "50.000".

## Config

### Main config: `~/.config/wallet/config.toml`

```toml
[database]
path = "~/.local/share/wallet/wallet.db"

[display]
currency = "IDR"
date_format = "2006-01-02"
first_day_of_week = "monday"

[ai]
json = true

[defaults]
account = ""
```

If the file doesn't exist, `Load("")` returns defaults — the app works without user configuration.

### Exchange rates: `~/.config/wallet/rates.toml`

```toml
base_currency = "IDR"
[rates]
USD = 15800
```

Rates represent 1 unit of foreign currency in base currency minor units.

## Testing

- **100% coverage enforced** via `make coverage-check` and CI (`go-quality.yml`)
- **`internal/testdb/`** provides in-memory SQLite with full migrations for service tests
- **`internal/gen/Querier`** interface enables mocking without a real database
- **Service override**: `getServiceOverride` in `internal/cli/helpers.go` allows CLI tests to inject a pre-configured service
- **Rate config override**: `service.SetTestRateConfig()` swaps `LoadRates`/`SaveRates` for in-memory testing
- **`coverignore.txt`** lists infrastructure-level branches (OS errors, data integrity guards) excluded from the coverage gate
- Each domain has its own `_test.go` file alongside the implementation

## sqlc Code Generation

Config: `sqlc.yaml`

```yaml
schema: internal/db/migrations
queries: internal/query
output: internal/gen (package "gen")
options: json_tags, db_tags, empty_slices, result_struct_pointers, emit_interface
```

After modifying `.sql` files in `internal/query/` or `internal/db/migrations/`, run `make sqlc-gen`. Never edit `internal/gen/` files manually.

## Pre-Commit Hook

`.githooks/pre-commit` blocks direct commits to `main`. Always work on a feature branch.
